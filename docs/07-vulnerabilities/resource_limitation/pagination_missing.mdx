# Pagination missing

## Description

The API returned a substantial amount of data.
A query pagination might be missing.


## Remediation

Implement pagination on the specified queries.



### GraphQL Specific

<details>
    <summary>Apollo</summary>

Implement pagination in your Apollo server by using cursor-based pagination, which involves passing a cursor that points to the last item in the previous fetch.

</details>

<details>
    <summary>Yoga</summary>

Use the Prisma client's pagination capabilities to implement cursor-based pagination in your GraphQL Yoga server.

</details>

<details>
    <summary>Awsappsync</summary>

In AWS AppSync, implement pagination by using the 'limit' and 'nextToken' arguments in your resolvers to manage the data returned.

</details>

<details>
    <summary>Graphqlgo</summary>

In GraphQL Go, implement pagination by using the 'after' and 'first' arguments in your schema to fetch slices of data.

</details>

<details>
    <summary>Graphqlruby</summary>

Use the 'graphql-ruby' gem's built-in pagination features to add cursor-based pagination to your queries.

</details>

<details>
    <summary>Hasura</summary>

Leverage Hasura's built-in support for cursor-based pagination using the 'limit' and 'offset' fields in your GraphQL queries.

</details>


### REST Specific

<details>
    <summary>Asp.net</summary>

Implement pagination in your ASP.NET Core Web API by adding parameters for page number and page size in your action methods.

</details>

<details>
    <summary>Ruby on rails</summary>

Use the 'will_paginate' or 'kaminari' gem to add pagination to your Rails API endpoints.

</details>

<details>
    <summary>Next.js</summary>

In Next.js API routes, implement pagination by parsing query parameters for page and limit to slice the data returned.

</details>

<details>
    <summary>Laravel</summary>

Use Laravel's built-in pagination methods to easily paginate the results of a query in your API.

</details>

<details>
    <summary>Express.js</summary>

In Express.js, use query parameters to implement pagination in your API routes by limiting the results returned from the database.

</details>

<details>
    <summary>Django</summary>

Utilize Django's built-in Paginator class to add pagination to your Django REST Framework views.

</details>

<details>
    <summary>Symfony</summary>

Implement pagination in your Symfony API by using the 'Pagerfanta' bundle or similar to paginate query results.

</details>

<details>
    <summary>Spring boot</summary>

In Spring Boot, use the Pageable interface to handle pagination automatically in your REST controllers.

</details>

<details>
    <summary>Flask</summary>

Use Flask's request object to parse pagination parameters and SQLAlchemy's paginate method to return paginated results.

</details>

<details>
    <summary>Nuxt</summary>

In Nuxt.js, you can create a server middleware or API route that accepts pagination parameters and returns a sliced array of data.

</details>

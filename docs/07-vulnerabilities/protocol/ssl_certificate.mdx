# SSL Certificate

## Description

The SSL certificate found on the website is no longer valid. This is most probably due to the fact that the SSL certificate is expired.


## Remediation

Purchase a new SSL certificate.



### REST Specific

<details>
    <summary>Asp.net</summary>

Renew the SSL certificate through your certificate provider and update it on your server. In IIS Manager, bind the renewed certificate to your website.

</details>

<details>
    <summary>Ruby on rails</summary>

Update your SSL certificate in the server configuration. For Nginx or Apache, place the new certificate files in the appropriate directory and update the server block or virtual host configuration.

</details>

<details>
    <summary>Next.js</summary>

Next.js typically relies on a Node.js server or hosting provider for SSL. Update your SSL certificate with your hosting provider, or if using a custom server, update the certificate in your server configuration.

</details>

<details>
    <summary>Laravel</summary>

Renew and install the SSL certificate on your web server. For Apache or Nginx, update the SSL configuration block with the new certificate files.

</details>

<details>
    <summary>Express.js</summary>

Renew the SSL certificate and update the server configuration to use the new certificate files.

</details>

<details>
    <summary>Django</summary>

Renew your SSL certificate and configure your web server (e.g., Nginx, Apache) to use the new certificate files.

</details>

<details>
    <summary>Symfony</summary>

Renew the SSL certificate and update your web server configuration to include the new certificate files.

</details>

<details>
    <summary>Spring boot</summary>

Renew the SSL certificate and configure it in your Spring Boot application properties or yml file.

</details>

<details>
    <summary>Flask</summary>

Renew the SSL certificate and update the Flask application to reference the new certificate files when starting the server.

</details>

<details>
    <summary>Nuxt</summary>

Nuxt.js applications typically rely on a Node.js server or hosting provider for SSL. Update your SSL certificate with your hosting provider, or if using a custom server, update the certificate in your server configuration.

</details>

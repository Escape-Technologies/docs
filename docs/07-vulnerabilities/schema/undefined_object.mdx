# Undefined objects

## Description

Undefined objects are objects that use the built-in GraphQL object type instead of referencing a custom one.
They can be at the root of security issues due to their unstructured nature.


## Remediation

Enforce strong typing in your schema objects.



### GraphQL Specific

<details>
    <summary>Apollo</summary>

Ensure that all types in your Apollo GraphQL schema are explicitly defined. Avoid using the generic GraphQLObjectType without proper field definitions to prevent unstructured data vulnerabilities.

</details>

<details>
    <summary>Yoga</summary>

Define all object types explicitly in your GraphQL Yoga schema to avoid security issues related to unstructured data.

</details>

<details>
    <summary>Awsappsync</summary>

Use AWS AppSync's schema definition language to define your types explicitly, and avoid using undefined objects to maintain structured data.

</details>

<details>
    <summary>Graphqlgo</summary>

In your Go application using GraphQL, define all types explicitly and avoid using undefined objects to ensure data integrity.

</details>

<details>
    <summary>Graphqlruby</summary>

In GraphQL-Ruby, define your types explicitly using Ruby classes to avoid the use of undefined objects and ensure data structure.

</details>

<details>
    <summary>Hasura</summary>

When using Hasura, ensure that all GraphQL types are explicitly defined in your schema, and avoid using undefined objects to prevent security vulnerabilities.

</details>

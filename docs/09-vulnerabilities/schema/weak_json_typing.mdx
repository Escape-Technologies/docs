# Weak JSON typing

## Description

Inputable JSON scalars are an arbitrary scalar type that allow users to return JSON objects from the schema.
It is a weak typing **bad practice** and may represent an unhandled data leak risk for your application.


## Remediation

When possible, use typed input objects instead.



### GraphQL Specific

<details>
    <summary>Apollo</summary>

Ensure strong typing in your schema by specifying exact types for all fields. Avoid using generic JSON scalar types that can lead to unhandled data and potential data leaks.

</details>

<details>
    <summary>Yoga</summary>

Define precise GraphQL types and avoid using arbitrary JSON types to prevent unhandled data leaks. Use the GraphQL schema to enforce type safety.

</details>

<details>
    <summary>Awsappsync</summary>

Utilize AWS AppSync's built-in scalar types or define custom scalar types to ensure that your GraphQL API is strongly typed and secure.

</details>

<details>
    <summary>Graphqlgo</summary>

In Go, define your GraphQL types strictly and avoid using interfaces{} as a way to accept any JSON object. This will help maintain type safety in your application.

</details>

<details>
    <summary>Graphqlruby</summary>

Use Ruby's GraphQL type system to define fields explicitly and avoid untyped scalars. This ensures that the data conforms to the expected structure.

</details>

<details>
    <summary>Hasura</summary>

Leverage Hasura's ability to create strict GraphQL types and avoid using JSON as a scalar type to prevent data leaks and ensure type safety.

</details>

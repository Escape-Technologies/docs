# Source code disclosure

## Description

The source code for the current page was disclosed by the web server.


## Remediation

Ensure that `.git`, `.svn`, `.htaccess` metadata files are not deployed to the web server or application server, or cannot be accessed.



### GraphQL Specific

<details>
    <summary>Apollo</summary>

Ensure that the Apollo Server is configured to not expose sensitive files. Use the 'playground' option only in development environments and disable introspection and the GraphQL Playground in production.

</details>

<details>
    <summary>Yoga</summary>

Configure GraphQL Yoga to disable the playground in production environments and restrict access to the schema by disabling introspection.

</details>

<details>
    <summary>Awsappsync</summary>

For AWS AppSync, restrict the API access using AWS IAM roles or API keys, and ensure that the schema is not publicly accessible without proper authentication.

</details>

<details>
    <summary>Graphqlgo</summary>

In the GraphQL Go implementation, disable introspection queries and the interactive editor in production by setting appropriate configuration flags.

</details>

<details>
    <summary>Graphqlruby</summary>

For GraphQL-Ruby, remove the GraphiQL IDE from production and ensure that introspection is disabled to prevent source code disclosure.

</details>

<details>
    <summary>Hasura</summary>

Secure Hasura GraphQL endpoints by using environment variables to disable the console and introspection features in production.

</details>

## Configuration

> Identifier: `information_disclosure/code`

### Options

- **size_threshold** : The threshold size indicating whether a response is small or not.
  - **diff_threshold** : The percentage by which 2 responses can differ and still be considered identical.
  - **small_response_diff_threshold** : The percentage by which 2 small responses can differ and still be considered identical.
